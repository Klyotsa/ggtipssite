======== ИНСТРУКЦИЯ ПО ХОСТИНГУ САЙТА GGTIPS ========

В данном архиве содержатся все необходимые файлы для
размещения сайта GGTips на хостинге.

== ВАЖНО! ИНСТРУКЦИЯ ПО РАСПАКОВКЕ ==

При распаковке архива на хостинг:
1. ОБЯЗАТЕЛЬНО сохраняйте оригинальную структуру папок и файлов
2. РАСПАКОВЫВАЙТЕ все файлы архива НАПРЯМУЮ в корневую директорию вашего хостинга
3. НЕ создавайте дополнительные подпапки при распаковке
4. УБЕДИТЕСЬ, что скрытые файлы (начинающиеся с точки, например .htaccess) также загружены

== ИНСТРУКЦИЯ ПО РАЗМЕЩЕНИЮ ==

1. ОБЫЧНЫЙ ХОСТИНГ (Apache/Nginx)
   - Распакуйте ВЕСЬ архив целиком в корневую директорию вашего хостинга
   - Убедитесь, что файл .htaccess корректно загружен (он может быть скрыт)
   - Проверьте правильность загрузки всех изображений и медиафайлов
   - Проверьте, что хостинг поддерживает единую точку входа (SPA)

2. NETLIFY
   - Создайте новый сайт на Netlify
   - Распакуйте архив на компьютере
   - Загрузите ВСЕ распакованные файлы через интерфейс Drag & Drop
   - Или подключите репозиторий и установите команду сборки: npm run build

3. VERCEL
   - Создайте новый проект на Vercel
   - Загрузите ВСЕ распакованные файлы
   - Или подключите репозиторий GitHub

== ПРОВЕРКА ПОСЛЕ РАЗМЕЩЕНИЯ ==

После загрузки файлов на хостинг обязательно проверьте:
1. Корректно ли отображаются все изображения и медиафайлы
2. Работает ли навигация по сайту (переходы между разделами)
3. Открываются ли страницы по прямым ссылкам (например, /services)

== РЕШЕНИЕ ПРОБЛЕМ С ИЗОБРАЖЕНИЯМИ ==

Если изображения не отображаются:
1. Проверьте, что все файлы из архива загружены в корень сайта
2. Убедитесь, что сохранены оригинальные имена файлов 
3. Проверьте права доступа к файлам (обычно 644 или 755)
4. Проверьте в консоли браузера (F12) ошибки загрузки ресурсов и пути к файлам

== ВАЖНЫЕ ФАЙЛЫ ==

- index.html - главная страница (точка входа)
- .htaccess - настройки Apache для правильной работы сайта
- assets/ - скрипты, стили и оптимизированные изображения
- Все изображения и медиафайлы находятся в корне и подпапках

== ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ ==

- Поддержка SPA (Single Page Application)
- Поддержка всех типов медиафайлов (jpg, png, gif, mp4, webm и др.)
- Правильные MIME-типы для файлов
- Рекомендуемые настройки кэширования для статических ресурсов

При возникновении проблем обратитесь к полной документации
или свяжитесь с разработчиками.
